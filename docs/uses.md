# Uses

> The tools, technologies, and setup I use daily.

<br/>

---

## Development

| Category | Tool | Why |
|:---|:---|:---|
| Editor | **VS Code** | Extensible, fast, and configurable. The right extensions make it work for any language. |
| Terminal | **macOS Terminal / iTerm2** | Clean, fast, gets out of the way. |
| Version Control | **Git + GitHub** | The standard. GitHub Actions and GitHub Pages add automation and hosting for free. |
| API Testing | **Postman** | Quick API testing and documentation. |
| Package Manager | **npm / pnpm** | pnpm for speed when it matters, npm for compatibility. |

---

## Frameworks & Libraries

| Purpose | Tool | Why |
|:---|:---|:---|
| Frontend Framework | **Next.js** | Server components, file-based routing, and seamless Vercel deployment. The best React framework for production. |
| Styling | **Tailwind CSS** | Utility-first CSS that keeps styling co-located with components. Fast iteration, consistent design. |
| UI Components | **shadcn/ui** | Beautifully designed, accessible components that you own and can customize. |
| Backend | **Firebase** | Auth, Firestore, hosting, functions — everything needed to move fast. |
| Backend (Alt) | **Supabase** | When I need PostgreSQL, real-time subscriptions, or row-level security. |
| Deployment | **Vercel** | Deploy in seconds. Preview deployments and edge functions are excellent. |

## AI & Machine Learning

| Purpose | Tool | Why |
|:---|:---|:---|
| ML Framework | **Scikit-Learn** | Practical, well-documented, and covers most real-world ML needs. |
| Data | **Pandas, NumPy** | The foundation of any data workflow in Python. |
| Notebooks | **Jupyter** | Best environment for exploratory analysis and ML experiments. |
| LLM APIs | **Gemini API** | Powerful multimodal capabilities. |
| Deep Learning | **TensorFlow, PyTorch** | TensorFlow for production, PyTorch for research and prototyping. |
